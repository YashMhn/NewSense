# news-scraper

A production-grade news scraper pipeline built in Python. Discovers articles from RSS feeds and sitemaps, extracts content through a three-layer fallback chain, and stores everything in SQLite with automatic deduplication.

Built as a Data Engineering portfolio project — designed to grow from a manual script into a fully scheduled, orchestrated pipeline.

---

## How it works

```
Discovery          Extraction              Storage
─────────          ──────────              ───────
RSS Feeds    ──┐   Trafilatura  (primary)
               ├──► Newspaper4k (fallback) ──► SQLite (deduplicated)
Sitemaps     ──┘   Goose3      (fallback)      + optional CSV export
```

Articles that fail all three extractors are logged to `data/failed_urls.log` for review.

---

## Three-stage progression

| Stage | File | How to run | When to use |
|---|---|---|---|
| 1 | `main.py` | `uv run python main.py` | Manual, one-off runs |
| 2 | `scheduler.py` | `uv run python scheduler.py` | Daily automation on any OS |
| 3 | `dags/` | Airflow on Linux/WSL2 | Production orchestration |

Each stage is fully functional on its own. Start at Stage 1, graduate to Stage 3 when ready.

---

## Features

- RSS feed + sitemap discovery with automatic deduplication
- Three-layer extractor fallback (Trafilatura → Newspaper4k → Goose3)
- Per-extractor retry logic for transient network failures
- Concurrent scraping via `ThreadPoolExecutor` (~5x faster than sequential)
- SQLite storage — URL-based deduplication across every run
- Rotating `failed_urls.log` — never grows unbounded
- Extractor breakdown stats per run
- Optional CSV export for quick inspection
- 24-test suite covering discoverer, scraper, and database modules
- All config in one place — `config.py`

## Setup

> Requires [uv](https://docs.astral.sh/uv/) — a fast Python package manager.

```bash
git clone https://github.com/YOUR_USERNAME/news-scraper.git
cd news-scraper
uv sync
```

`uv sync` reads `pyproject.toml`, creates a virtual environment, and installs all dependencies in one step. No manual `pip install` or venv activation needed.

---

## Running

### Stage 1 — manual run

```bash
uv run python main.py
```

### Stage 2 — daily schedule

```bash
uv run python scheduler.py
```

Runs immediately on startup, then daily at `RUN_TIME` (default `08:00`). Keep it alive in a `tmux` or `screen` session on a server. All settings are in `config.py`.

### Stage 3 — Airflow (Linux / WSL2 only)

> Airflow does not support Windows natively. Run via WSL2 or a Linux server.

```bash
uv add apache-airflow
uv run airflow standalone          # creates ~/airflow/ on first run, Ctrl+C when ready

export AIRFLOW__CORE__DAGS_FOLDER=$(pwd)/dags
uv run airflow standalone          # now picks up news_pipeline_dag
# open http://localhost:8080 → enable: news_scraper_pipeline
```

### Tests

```bash
uv run pytest tests/ -v
```

## Configuration

Everything is in `config.py` — one file, no hunting across multiple scripts.

```python
# Discovery
RSS_FEEDS      = ["https://feeds.bbci.co.uk/news/rss.xml", ...]
SITEMAP_SITES  = ["https://www.thehindu.com", ...]
MAX_PER_SOURCE = 10

# Scraping
REQUEST_DELAY  = 1.0    # seconds between requests
MAX_WORKERS    = 5      # concurrent threads
MAX_RETRIES    = 2      # retries per extractor before fallback
MIN_TEXT_LENGTH = 100   # chars — shorter articles are skipped

# Scheduler
RUN_TIME = "08:00"      # 24-hour local time

# Output
EXPORT_CSV = True       # write CSV alongside the database
```

## Extraction fallback chain

```
URL
 │
 ├── 1. Trafilatura    F1: 0.958 — fastest, highest accuracy
 │        success → done
 │        fail    → retry up to MAX_RETRIES, then next layer
 │
 ├── 2. Newspaper4k    NLP extras: keywords, summary
 │        success → done
 │        fail    → retry, then next layer
 │
 ├── 3. Goose3         Strong on Indian/Asian news sites
 │        success → done
 │        fail    → log to data/failed_urls.log
```

## Database schema

```sql
CREATE TABLE articles (
    id           INTEGER PRIMARY KEY AUTOINCREMENT,
    title        TEXT,
    authors      TEXT,
    date         TEXT,
    summary      TEXT,
    text         TEXT,
    tags         TEXT,
    url          TEXT UNIQUE,       -- deduplication key
    source       TEXT,
    language     TEXT,
    extracted_by TEXT,
    scraped_at   TEXT,
    created_at   TEXT DEFAULT (datetime('now'))
);
```

## Packaging with uv

This project uses `uv` for dependency management. To replicate the setup from scratch:

```bash
# Initialise — auto-generates pyproject.toml
uv init --no-workspace

# Add dependencies — each command updates pyproject.toml automatically
uv add trafilatura newspaper4k goose3 feedparser beautifulsoup4 requests lxml schedule

# Dev dependencies
uv add --dev pytest

# Lock exact versions for reproducible installs
uv lock

# Anyone cloning the repo just runs:
uv sync
```

To export a traditional `requirements.txt`:

```bash
uv export --format requirements-txt > requirements.txt
```
